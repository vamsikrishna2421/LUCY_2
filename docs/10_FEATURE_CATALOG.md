# LUCY 1.0 — Feature Catalog (Parity Contract)

> The exhaustive, logic-level inventory of every distinct capability in frozen Lucy 1.0.
> Source of truth: `C:\Users\vamsy\Documents\lucy\app\src` (+ `app\App.tsx`).
> **One row per distinct capability.** Behavior is described in plain language, not code.
> This is the parity contract: a capability not catalogued here is one the 2.0 rebuild may silently lose.
>
> **Columns:** ID · Module/file · Feature · Trigger · Inputs · Behavior · Outputs/side effects · Dependencies · Visibility · Parity status
> **Visibility:** `visible-used` (user sees & touches routinely) · `visible-rare` (visible but seldom hit) · `invisible` (background jobs, silent fallbacks, migrations, dedup, conditional paths, edge handlers).
> **Parity status:** default `preserved` for every row.
