## PART ONE: THE STORY WE ARE TELLING

### The Big Idea (Memorise This)

> "You said it once. Three days later, your phone remembered it for you."

That's the whole product in one sentence. Every demo beat, every example, every tap on screen must come back to this. You are not showing a note-taking app. You are showing a **private AI second brain that organises your life while you live it** — no folders, no tags, no maintenance, no setup.

### The Human Problem (Tell This First)

Here is the opening story you tell — do not show the phone yet:

> *"I want you to think about the last time you had a brilliant idea in the shower, or in the car, or mid-conversation — and then completely forgot it by the time you sat down to write it down. Or the time a colleague said something important in a meeting and you thought 'I'll remember that' — and you didn't. Or the expense you forgot to log, the follow-up you promised to send, the reminder you never set.*
>
> *We live in a world where our most important thoughts arrive at the worst possible moments. Note apps demand structure you don't have time for. Voice memos pile up unlistened. Calendars only capture what you already planned.*
>
> *What if there was a system that just... listened? That turned your raw, messy, unstructured thoughts into organised memory — automatically. That remembered who you talked to, what you spent, what you promised, what you were worrying about — and then surfaced it back to you exactly when you needed it.*
>
> *That's LUCY."*

**Beat.** Show the phone.

### SCENE 10 — Technical Architecture (17:00 – 18:30)

**[Screenshare: Optional — architecture diagram or stay on app.]**

> *"Under the hood — for the technical judges:*
>
> *React Native with Expo, running on iOS and Android. SQLite with SQLCipher encryption as the data layer — not a cloud database. Every capture is in a durable processing queue with exponential backoff, automatic retry, and now — billing-aware error handling that stops retrying when you're out of credits.*
>
> *AI extraction uses a structured JSON schema with 20+ fields: tasks, expenses, ideas, reminders, places, people, open loops, follow-ups, mood, clarifications, memory gaps, and — new in the latest build — an action detection field that powers the automation engine.*
>
> *The model is configurable. Use Claude Sonnet for richer extractions. Switch to GPT-4o for speed. The routing layer handles both providers transparently — one key per provider, same UX.*
>
> *Semantic search uses vector embeddings — OpenAI Ada — stored locally in the same SQLite database. Hybrid BM25 + cosine similarity + temporal recency weighting. Queries run in under 300ms.*
>
> *The knowledge graph is rebuilt on every organisation run: entities, connections, insights — all derived from your raw captures, with evidence counts and confidence levels. Nothing is invented. Everything has a source.*
>
> *Background processing uses Expo Background Task — iOS and Android. Processes one capture per background window, respects battery constraints, and shows you exactly when it last ran."*

## PART SEVEN: THE PRODUCT — COMPLETE FEATURE INVENTORY

*(For judges who want to go deep. Have this on a second screen.)*

### Capture Layer
- Text input (any length, any language)
- Voice via SFSpeechRecognizer (iOS on-device STT) or Whisper fallback
- Share extension (share any text/URL/file from any iOS app into LUCY)
- Shared `.md`/`.txt` files read and ingested automatically
- Per-capture privacy toggle
- Real-time processing queue with status: Queued → Organizing → Remembered / Retrying
- Billing-aware retry: billing/quota failures are terminal (no runaway re-spend)
- Hourly AI call cap with configurable limit (cost guard)

### Extraction Engine
- 20+ field structured JSON schema extracted by Claude or GPT
- Fields: title, summary, note_type, detected_language, privacy_level, tasks, expenses, ideas, places, interests, people, reminders, open_loops, follow_ups, mood, clarifications, memory_gaps, tags, detected_action
- Multi-list splitting: "buy milk, eggs, bread" → 3 tasks automatically
- Day-journal splitting: one paragraph → multiple per-event timeline memories
- Multi-date journal: 90-day log → one capture per day with correct historical timestamps
- Intra-extraction dedup: "drink water" repeated 15x → deduplicated to 1 task
- Reprocess: purges prior derived data before re-running (no duplicates)

### Timeline
- Chronological, grouped by day
- Mood colour dot per card (positive=green, neutral=grey, stressed=amber, frustrated=red)
- Expand/collapse key points
- Failed capture surfaces amber "Couldn't organize — tap ⋯ to retry" with error detail
- Three-dot menu: Correct, Reprocess, Delete
- Delete cascades: removes from tasks, ideas, expenses, embeddings, brain
- Reprocess shows cost warning for long entries
- "LUCY can do this" inline banner for LLM-detected action commands
- Quick-capture bar at top of Timeline
- Semantic search (hybrid BM25 + vector + temporal)

### Tasks (Board)
- Auto-categorised: Grocery List, Daily Habits, Work & Deadlines, Calls & Messages, Health, Personal, General
- Animated completion with strikethrough
- Undo
- Scroll-fixed sheet (reliably scrollable even with 50+ items)
- Quick-add within a category
- Edit / delete per task
- Done Today section

### Ask
- Free-form conversational query
- Hybrid retrieval: semantic + keyword + entity + temporal
- Source citation chips ("from your memory" with capture title)
- Live device context injected: battery, timezone, time, upcoming calendar events
- Thread history
- Automation detection: imperative commands surface confirmation card instead of chat bubble
- Insights panel: 6 AI-generated questions from your actual patterns, tap to reveal answer

### Automation Engine
- LLM-detected: `detected_action` extracted from every capture's structured output
- Falls back to regex fast-path for instant UX before async extraction completes
- Actions: timer, call (contact lookup), navigate (Maps), play (Spotify/Apple Music), remind (iOS Reminders), message (SMS/iMessage), event (Calendar), shortcut (iOS Shortcuts)
- Always saves the capture as a memory even if the action fires
- Confirmation required before execution — never auto-executes
- Android: correct intent extras (`i.` integer, `B.` boolean flags)
- iOS: `LSApplicationQueriesSchemes` for all supported URL schemes

### Proactive Intelligence
- Morning brief (7-9am): overdue items + open loops + mood trend + relationship gaps → AI-synthesised 120-word message
- Weekly insight (Sunday 6-9pm): week in review → 2-3 sentence pattern reflection
- On This Day: same-date memories from past years → notification + Now card
- Pre-meeting brief (30 min before): attendee relationship history + relevant captures → 2-sentence notification
- Post-meeting prompt (15 min after): "capture your notes before they fade"
- Progress check-ins (every 2h, optional): nudge to log updates

### Brain (Library)
- Tabs: Todos · Ideas · Expenses · People
- People tab: everyone mentioned, mention count, last seen, follow-ups

### Settings / Diagnostics
- Model dropdown: OpenAI (gpt-4o-mini / gpt-4o / gpt-4.1 / gpt-5 series) + Anthropic (Haiku / Sonnet / Opus)
- Remote intelligence toggle (OpenAI) + Claude API key input
- Queue diagnostics: Queued / Organizing / Will retry / Remembered / Archived counts
- PROCESSING WITH panel: active model + remote ready/unavailable status
- Cost guard: hourly AI call cap, ON/OFF toggle, 60/120/240/500/hr cycle
- Recent errors: collapsible error log (bounded at 100 rows) with Clear button
- Background organizing toggle + last run time
- Export as JSON
- Reprocess all memories (warns about per-memory AI cost before proceeding)
- Delete all memories (FK-safe two-phase delete)

### Privacy
- SQLCipher encrypted SQLite (hardware-level key in SecureStore)
- No LUCY account, no LUCY server
- Per-capture privacy toggle
- Ideas always marked private
- Error log, call log, embeddings — all local

### CI / Build
- Codemagic pipeline (Mac mini M2)
- `expo prebuild` → Pods → Build IPA → TestFlight
- Build number auto-incremented from `$BUILD_NUMBER + 100` (prevents App Store Connect collision)
- No manual version bumps required
